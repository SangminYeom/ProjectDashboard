# Asana 스타일 UI/UX 개편 설계

작성일: 2026-07-10

## 배경 및 목표

현재 대시보드는 Apple 계열 비주얼(SF Pro, 애플 블루 `#0071e3`, 두꺼운 그림자 카드,
중앙 정렬 `max-width:1100px`, 사이드바 없음)로 되어 있다. 화면이 옛돼 보이고 프로젝트
간 이동이 불편하다는 문제가 있다.

Asana의 UI/UX를 참고해 **레이아웃 뼈대부터** 현대화한다. 기능은 그대로 두고 UI/레이아웃만
바꾼다.

## 확정된 결정

1. **범위**: 비주얼 리프레시가 아니라 구조까지 Asana식으로 (좌측 사이드바 + 뷰 탭).
2. **사이드바**: 프로젝트별 상태 표시 포함(시안 2). 접기 없이 **고정**.
3. **강조색**: Asana 코랄 (`#D85A30` 계열).
4. **상세 화면**: Asana식 뷰 탭 `개요 · 과제 · 운영 · 쟁점`. 쟁점은 프로젝트 전체에서
   모아 한 탭으로.
5. **다크 모드**: 없음. 라이트 모드 전용 유지.

## 전체 레이아웃 (앱 셸)

중앙 정렬 컨테이너를 없애고 **좌측 고정 사이드바 + 우측 메인 영역**의 2단 구조로 전환한다.

- 사이드바 폭: 약 240px, 고정. 메인 영역은 남는 폭을 채우되 내부 콘텐츠는 가독성을 위해
  적절한 최대폭을 둔다.
- 화면 전체 배경은 옅은 회색(현재 `#f5f5f7` 계열 유지), 사이드바와 카드는 흰색 계열.

### 사이드바 구성 (시안 2)

위에서 아래로:

1. 상단 유닛명 헤더 — `고객가치혁신유닛`
2. **전체 개요** 항목 (홈 아이콘) — 클릭 시 프로젝트 카드 그리드(홈)로 이동
3. `프로젝트` 섹션 라벨
4. 프로젝트 목록 — 각 행: `색점 + 프로젝트명 + 상태 배지`
   - 상태 배지 규칙: 미해결 쟁점이 있으면 `쟁점 N`(위험색), 없으면 전체 진척률 `NN%`,
     운영 상태가 주의/위험이면 해당 상태 텍스트. (표시 우선순위는 구현 단계 계획에서 확정)
5. **새 프로젝트** 항목 (+ 아이콘)

현재 열려 있는 프로젝트(또는 전체 개요) 항목은 코랄 톤 배경으로 하이라이트한다.

## 화면 구성

### 전체 개요 (홈)

지금의 프로젝트 카드 그리드를 **유지하되** 새 비주얼 시스템으로 다시 스타일링한다.
사이드바의 "전체 개요"를 클릭하면 진입한다. 카드는 얇은 테두리 + 옅은 그림자의
평평한 스타일로 바꾸고, 카드 상단에 프로젝트 색점을 노출한다. 카드 내부 요약(KPI·과제·
운영·쟁점)의 정보 구성은 유지한다.

### 프로젝트 상세

상단에 `프로젝트 색점 + 프로젝트명`과 기간을 표시하고, 그 아래 **뷰 탭**을 둔다.
탭: `개요 · 과제 · 운영 · 쟁점`. 활성 탭은 코랄 밑줄로 표시한다.
헤더 우측의 기존 액션(PNG 내보내기, 수정, 삭제)은 유지한다.

- **개요 탭**
  - KPI를 상단 지표 카드 행으로 표시(수치형은 값 + 진행바, 정성형은 상태 텍스트).
  - 아래 2열로 `과제 진척 요약`(과제명 + 진행바 + %)과 `주요 쟁점 요약`(중요도 점 +
    쟁점 내용 + 상태)을 나란히 배치.
  - 기존 KPI 편집/드래그 기능은 유지(개요 탭 안에서 편집).

- **과제 탭**
  - 기존 중점수행과제 + 간트(타임라인)를 새 스타일로. 카드/트랙/마일스톤/오늘 마커 등
    구조와 기능은 유지하고 비주얼만 정리.

- **운영 탭**
  - 기존 운영업무 테이블을 새 스타일로.

- **쟁점 탭**
  - 프로젝트 내 **모든 과제의 쟁점을 하나의 목록**으로 모아 표시한다.
  - 각 쟁점 행에 **소속 과제명을 라벨**로 표시한다.
  - 쟁점 **추가** 시 "소속 과제 선택" 필드를 두고, 데이터는 기존과 동일하게 해당 과제의
    `issues` 배열에 저장한다(데이터 구조 변경 없음).
  - 미해결/해결 구분, 중요도(상/중/하), 상태(대기/진행중/해결) 등 기존 기능 유지.

## 비주얼 시스템

- **강조색(accent)**: Asana 코랄. 기본 `#D85A30`, hover/진한 단계(`#993C1D`), 옅은 배경
  (`#FAECE7`) 포함. 버튼, 활성 탭 밑줄, 링크, 진행바, 포커스 링에 적용.
- **프로젝트 색점 팔레트**: 퍼플(`#7F77DD`)·틸(`#1D9E75`)·코랄(`#D85A30`)·블루
  (`#378ADD`)·핑크·앰버 등에서 프로젝트마다 하나를 지정. 사이드바·상세 헤더·홈 카드에서
  같은 색을 일관되게 사용한다. (프로젝트별 색상 지정 방식의 저장 위치는 구현 계획에서 확정)
- **카드**: 두꺼운 그림자 → **0.5px 테두리 + 아주 옅은 그림자**의 평평하고 깔끔한 스타일.
  모서리 반경은 카드 12px, 컨트롤 8px 수준.
- **상태 색**: 정상(초록)/주의(앰버)/위험(빨강), 쟁점 중요도(상=빨강, 중=앰버, 하=회색)는
  유지하되 코랄 기반 팔레트와 어울리게 톤 정리.
- **타이포·여백**: 현재 폰트 스택 유지, 위계·여백을 Asana식으로 정리(과한 대비 축소,
  일관된 리듬).

## 유지되는 것 (기능 불변)

- 로그인/로그아웃, 세션
- PNG 내보내기(스냅샷)
- 데이터 구조(`data/projects.json`) 및 백업
- 드래그 정렬(KPI·과제·간트·쟁점)
- KPI·과제·간트·운영·쟁점의 계산 로직 및 편집 동작

이번 작업은 **UI/레이아웃 재구성**이며, 위 기능들의 동작은 바뀌지 않는다.

## 영향 받는 주요 파일 (예상)

- `src/App.jsx` — 앱 셸(사이드바 + 라우팅/뷰 전환) 도입
- `src/pages/Home.jsx` — 전체 개요(카드 그리드) 재스타일
- `src/pages/Project.jsx` — 뷰 탭 구조(개요/과제/운영/쟁점)로 재구성, 개요 탭 신설
- `src/components/*` (KpiBar, Initiatives, Gantt, OperationsTable, IssueLog) — 새 스타일
  적용, 쟁점 집계 뷰 신설
- `src/styles.css` — 코랄 기반 토큰, 앱 셸/사이드바/탭/카드 스타일 전면 정리
- 신규: 사이드바 컴포넌트, 쟁점 집계 뷰 컴포넌트(구현 계획에서 구체화)

## 열린 세부 (구현 계획에서 확정)

- 사이드바 상태 배지 표시 우선순위(쟁점 vs 진척률 vs 운영 상태)
- 프로젝트별 색점 지정/저장 방식
- 라우팅 방식(현재 상태 기반 뷰 전환 유지 여부)
- 메인 영역 콘텐츠 최대폭
